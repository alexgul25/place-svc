# Place Service (place-svc)

Микросервис для проекта **Date Wishlist Hub**, отвечает за данные о местах, добавленных пользователями.  

Стек: `Go`  `gRPC`  `PostgreSQL`  `Kafka`  `Redis`

## Основные возможности

Все `.proto` контракты хранятся в репозитории :point_right: [protos](https://github.com/alexgul25/protos).

**Place Service** предназначен только для внутреннего межсервисного взаимодействия и не должен быть доступен пользователям напрямую. Все запросы от конечных пользователей принимает [Gateway Service](https://github.com/alexgul25/gateway-svc).

- Добавление нового места (метод `AddPlace` — вызывается через [Gateway Service](https://github.com/alexgul25/gateway-svc), для этого пользователю нужен JWT-токен). При успешном добавлении генерируется событие `place.created`, атомарно сохраняется в outbox-таблице и затем отправляется в `Kafka` фоновым процессором. Это гарантирует надёжную доставку события другим сервисам.

- Получение списка мест пользователя (метод `GetUserPlaces` — вызывается через [Gateway Service](https://github.com/alexgul25/gateway-svc), для этого пользователю нужен JWT-токен). Для снижения нагрузки на базу данных используется кэширование в `Redis`. При первом запросе данные загружаются из `PostgreSQL` и помещаются в кэш, повторные запросы возвращаются из кэша. Кэш инвалидируется при добавлении нового места пользователем.

Все запросы для **Place Service** должны передавать заголовок `x-service-name` — имя сервиса, вызывающего метод.  

Для методов, требующих идентификации через JWT токен, необходимо передавать заголовок `x-user-id`.
